# 项目开发计划

## 1 项目概述

### 1.1 应用背景

#### 场景：智慧农业温室监测与设备管理系统

随着精准农业和智能温室技术的发展，现代农业对环境监测和设备管理提出了更高的要求。本系统针对**农业科研温室**和**商业化植物工厂**设计，提供一体化的环境感知、设备监控和人员物资管理解决方案。

### 1.2 项目目标

- 实现符合 ISO/EC 14443 标准的读卡/写卡功能（样本标签、人员工牌）
- 集成多类型传感器数据采集（温度、光照、霍尔、振动）
- 构建稳定的串口通信与网络传输层
- 提供完整的后端 API 与前端可视化界面

### 2.2 各层职责

#### 感知层
| 模块 | 功能描述 | 应用场景 |
|:-----|----------|:---------|
| ISO/EC 14443 协议 | 实现非接触式智能卡通信协议，支持读卡、写卡、防碰撞、加密 | 植物样本 RFID 标签读写 |
| 串口模块 | 负责与硬件设备的串行通信，进行数据预处理 | 传感器节点数据上传、RFID 读卡器通信 |
| 传感器 | 采集环境数据：温度、光照、霍尔效应、振动，通过数模转换输出 | 温室环境监测、设备状态检测 |

#### 网络层
| 模块 | 功能描述 | 应用场景 |
|------|----------|----------|
| GUI 封装 | 提供图形化界面封装，统一用户交互入口 | 温室现场操作终端 |
| 串口控制 | 管理串口连接、配置、数据收发 | RFID 读卡器管理 |
| 数据提取/封装 | 从原始数据中提取有效信息并封装传输格式 | 传感器数模转换、RFID 数据解析 |
| 日志输出 | 记录系统运行日志 | 调试和追溯，故障排查 |
| 网页请求处理模块 | 处理 HTTP 请求，协调上下层工作 | Web 界面数据请求、API 调用 |

#### 应用层
| 模块 | 功能描述 | 应用场景 |
|------|----------|----------|
| 后端 API | 提供Flask API 接口，支持信息采集、后台管理、数据库管理 | 样本管理、设备管理 API |
| 前端 | 提供页面交互、后台界面、数据可视化功能 | 温室 Dashboard、样本查询、告警中心 |
| 数据库管理 | 数据存储、查询、备份、恢复 | 环境历史数据、实验记录、操作日志存储 |

## 3 开发计划

#### 第一阶段：需求分析与设计

完成需求分析，架构设计，具体功能的构建。

#### 第二阶段：感知层开发

1. 实现传感器数据读取与数模转换
2. 根据ISO/EC 14443协议，设计指令系统
2. 读卡写卡操作实现
3. 依据指令系统，实现串口通信

#### 第三阶段：网络层开发

1. GUI框架搭建
2. 指令编解码模块实现
3. 数据处理模块，同时完成日志系统
4. 网页请求模块实现，完成数据向上层传输

#### 第四阶段：应用层开发

1. 后端API开发
2. 数据库设计与实现
3. 前端界面开发，完成数据可视化
4. 后台管理系统实现
